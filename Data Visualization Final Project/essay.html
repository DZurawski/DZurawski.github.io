Essay

Hybrid Structure

I chose to use the interactive slideshow hybrid structure for my final project. The question that my presentation focuses on is: "In blitz chess, should you castle?"

Scenes

I chose to include two scenes within my presentation. Scene 1 displays probability of castling versus a player's elo rating. This scene helps answer my focus question by showing that high-level players tend to castle more frequently than low-level players. Scene 2 displays number of moves before castling versus the white player's average score. This scene shows that players who do not castle tend to score lower than players who do castle. It also shows that there is generally low correlation between number of moves before castling and player score.

The layout of each scene is a singular chart with clarifying information below it. This layout brings the chart to the user's full attention and provides detailed information below if the user desires to read it. At the very bottom are buttons that progress to the next scene.

Annotations

In the line graph, I chose to use text annotations to highlight how the probability of castling is correlated with a player's elo. I chose to include the approximate probability of castling within each annotation to show how significant the percentage point change is (from ~50% to ~97%). Also, if users mouseover a data point, more information about that data point is provided as a pop-up annotation. I chose to make these annotations as pop-ups because they provide specific information that can be intrusive if these annotations were always visible.

In the heat map, I chose to use text annotations to highlight how not castling is correlated with a lower score. Similarly with the line graph, hovering your mouse over a heat map square will display extra information through a pop-up annotation. This pop-up annotation is very heavy with information, so I chose to make it only appear if users request it through mousing over it. By doing this, the annotation is not obstructive to viewing the chart as a whole.

All annotations are cleared between scenes because the annotations are most relevant to the scene they are introduced at.

Parameters

Current Scene Parameter - The current scene parameter determines which scene the user is currently viewing. This parameter can be changed by the user through using the progression buttons at the bottom or through the navigation bar. The current scene parameter is displayed in the navigation bar, where the current scene's number has a white box border around it.

Dynamic Annotation Parameter - Users can mouse over data points and heat map rectangles to display specific information. The dynamic annotation parameter contains this information.

Triggers

Progression Button Trigger - To progress between scenes, I chose to use buttons that the user can click at the bottom. These buttons are labeled "previous" and "next", so it is clear to the user that these buttons progress the slideshow. They change the current scene parameter.

Navigation Bar Trigger - I also included a navigation bar at the top so that users can go back and view specific scenes after they have finished the slideshow. The navigation bar highlights which scene the user is currently on and mousing over the scene numbers highlights them, indicating to the user that they can click it.

Mouse Over Trigger - Users are able to mouseover data points to display specifying information. This changes the dynamic annotation parameter's content and display visibility. The viewer is aware of this because the scenes say to mouse over points to display more information.
